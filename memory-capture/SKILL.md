# Memory Capture — 记忆自动捕获

## 定位

对话过程中自动检测记忆信号，写入 `memory/pending-extract.md`，为归档做准备。

## 触发方式

### 主动捕获（对话中自动）

**检测信号类型**：

| 类型 | 关键词/模式 | 示例 |
|------|------------|------|
| 偏好表达 | "我喜欢"、"我讨厌"、"我一般"、"通常" | "我喜欢用 Python" |
| 明确决策 | "决定了"、"选定了"、"就这样"、"最终方案" | "我们决定用 Flask" |
| 教训反思 | "下次要注意"、"之前"、"这次"、"曾经" | "上次部署忘了重启" |
| 项目状态变化 | "完成了"、"失败了"、"放弃了" | "这个项目放弃了" |
| 工具/技术变化 | "换成"、"改用"、"迁移到" | "换成了 ChromaDB" |
| 上下文更新 | "其实"、"准确说"、"更正" | "其实我更常用 Mac" |

**写入时机**：
- 检测到信号 → 写入 pending（带 `### 主题：` 格式）
- 不打断对话，在后台追加
- 结束时（说"就到这里"）统一确认

---

### 收尾提醒（自动触发）

**收尾信号**：
- "就到这里"、"先这样"、"今天就到这里"、"先这样吧"
- 检测到 `/new` 或对话即将结束时

**提醒话术**：
> 🔔 发现 {N} 条待归档记忆，是否现在运行 `/memoryre` 归档？

---

## 数据格式

写入 `memory/pending-extract.md` 的内容：

```markdown
### 主题：Profile → 沟通偏好
**检测时间**：2026-04-11 10:20
**类型**：偏好表达
**内容**：
- 用户表示喜欢简洁的回复风格
- 不喜欢列表式的表达
```

---

## 操作（Actions）

### `capture` — 捕获记忆

输入：`content`（原始文本）、`type`（信号类型，可选，自动推断）

流程：
1. LLM 解析 content，提取记忆片段
2. 匹配主题（Profile / Decisions / Lessons / Projects / Tech Stack）
3. 追加到 pending-extract.md
4. 返回捕获确认

### `status` — 查看待归档数量

输入：无

流程：扫描 pending-extract.md，统计待归档条数

### `remind` — 收尾提醒

输入：无

流程：检测 pending 条数，超过阈值（≥1）则提醒归档

---

## 自动捕获集成（SOUL.md 行为规则）

在 SOUL.md 中新增：

**对话中**：每次回复后，检查本轮消息是否含记忆信号。如有，调用 `memory-capture`，`action=capture`。

**不拦截**：已经在讨论记忆归档时、主动触发 `/memoryre` 后 30 分钟内。
